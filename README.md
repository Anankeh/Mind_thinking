# Learning Architecture of Technical Learning

This project is an interactive visual model that represents how learning architecture turns complex technical systems into capability people can prove.

The model uses five public, product-free service lines:

- Service Operations
- Endpoint Management
- Backup and Recovery
- Security Operations
- Knowledge Systems

At the center is **Govern**, inspired by the NIST Cybersecurity Framework. Governance is treated as the place where standards, validation, role boundaries, evidence, and iteration stay coherent.

The visual shows three movements:

- Decisions and standards move outward.
- Evidence from practice returns inward.
- The learning architect moves across the system to keep methodology, content, operations, and capability aligned.

The three knowledge levels are represented with symbols instead of labels:

- ✳ Basic
- ◆ Administrator
- ● Advanced

This artifact is part of a public portfolio documenting my thinking process as a psychologist and learning architect: translating technical complexity into clear, usable, evidence-based learning systems.
